# 采集与浮窗方案（iPhone）

## 选定方案
- PiP 浮窗 + 打点辅助 + 实时建议
- 不抓包；允许录音与录屏；全员同意录音录屏

## 采集流程
1. 用户从系统录屏入口启动录屏并选择 App 广播扩展
2. 录屏包含：屏幕画面 + 麦克风音频
3. App 启动 PiP 浮窗显示关键状态信息与实时建议（只展示，不交互）
4. 通过“打点”记录关键节点（发言开始/投票开始/投票结束等）
5. 实时建议在录制过程中生成并推送到 PiP（轻量级）
6. 赛后回到 App 前台进行解析与校对（高精度）

## 打点方式（可并存）
- 耳机线控/蓝牙按钮
- 音量键监听（仅作标记，不改变系统音量）
- 快捷指令触发打点
注：音量键打点在 iOS 上仅在 App 前台有效，后台录制时需备用方式（手动或快捷指令）。

## 解析策略
- 音频：VAD 切分发言段落 -> ASR 转写 -> 术语纠错
- 画面：投票画面 OCR -> 票型结构化
- 打点：用作切片边界与阶段对齐

## 实时建议策略
- 目标：低延迟提示当前阶段的关键行动与风险点
- 输入：打点事件 + 低精度 ASR 关键词 + 已知公开信息
- 输出：仅提示性建议，不下“确定身份”结论
- 计算：在广播扩展内做轻量推断，复杂推断留给赛后

## PiP 浮窗内容规范
- 当前回合与阶段（夜/昼）
- 关键状态：警长、存活人数、已知公开信息
- 提示类信息：下一步待做事项（例如“标记投票开始”）
- 实时建议摘要：1-2 行可读提示
- 只读展示，不承载输入

## 约束
- iPhone 不支持任意 UI 悬浮，仅使用 PiP 视频窗口
- 不在后台执行交互式 UI
- 所有推断在本地完成，默认不上传
- 实时建议仅作为提醒，不替代赛后推断结论
